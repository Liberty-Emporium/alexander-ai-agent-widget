How Actions Work: When a user asks {{ agent.name }} to DO something (e.g. "create a claim", "add a room", "look up my order"), the agent detects the intent, picks the right action, fills in the parameters, and makes the API call to your app — then reports back what happened.

URL placeholders: Use {param_name} in URLs and body templates. Example: https://billy-floods.up.railway.app/claims/{claim_id}/room/add
Auth: The agent's OpenRouter API key is sent as Authorization: Bearer <key> unless you override it in Headers JSON.

{{ agent.name }}'s Self-Management API

Give this to {{ agent.name }} and he can create his own actions.

{{ agent.name }} already knows about this API — it's injected into his system prompt automatically. Share it with him in chat: "Willie, your API key is below. Use it to add new actions whenever you need them."
POST — Add / Update an Action
POST {{ request.host_url.rstrip('/') }}/agent/{{ agent.id }}/actions/api Authorization: Bearer {{ agent.api_key }} Content-Type: application/json { "name": "action_name", "description": "When to use this action", "method": "POST", "url": "https://your-app.com/endpoint", "body": {"param": "{param}"} }
GET — List Current Actions
GET {{ request.host_url.rstrip('/') }}/agent/{{ agent.id }}/actions/api Authorization: Bearer {{ agent.api_key }}
DELETE — Remove an Action
DELETE {{ request.host_url.rstrip('/') }}/agent/{{ agent.id }}/actions/api Authorization: Bearer {{ agent.api_key }} Content-Type: application/json {"name": "action_name"}
Example: Tell {{ agent.name }} in chat:
"⚡ Willie, I need you to be able to update insurance info on a claim. Your API key is {{ agent.api_key[:12] }}••• and your actions endpoint is {{ request.host_url.rstrip('/') }}/agent/{{ agent.id }}/actions/api — go ahead and add that action."

Willie will call the API himself and the new action will appear on this page instantly.
